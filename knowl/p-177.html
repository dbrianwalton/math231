This idea perhaps feels weirder when we have a function defined in abstract, such as \(f(x)=x^2-3x\) and then write something like \(f(2x+1)\text{.}\) In one sense, it might feel like we are saying \(x\) equals \(2x+1\text{.}\) This is where it is helpful to think of \(f\) as the rule of what we do to the input (and not what we do to \(x\)). We need to square the input \((2x+1)\) and then we need to subtract from that three times the input \((2x+1)\text{.}\)

\begin{equation*} f(2x+1) = (2x+1)^2 - 3(2x+1)\text{.} \end{equation*}

If we compare this to the original formula,

\begin{equation*} f(x) = x^2 - 3 x\text{,} \end{equation*}

we can see that we are substituting the input formula in place of the input variable. Because the input formula represents a single value, we wrap it in parentheses as a group.
